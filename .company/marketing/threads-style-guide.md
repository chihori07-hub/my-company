# Threads専用 スタイルガイド
# ちほさんの分身として書くための完全ガイド

> ブランドトーン・NGワードは `CLAUDE.md` を参照。
> このファイルを読めば、実投稿を参照しなくても「ちほさんの文章」として書ける。
> 実投稿（campaigns/反応が良かったThreads投稿とテーマ.pdf）およびCSVデータ分析から導出。

---

## 発信の基本方針

| 項目 | 内容 |
|------|------|
| 読んだ人に持ってほしい印象 | 信頼できる専門家 |
| 主要プラットフォーム | Threads |
| 文体 | です・ます調（体言止め・短い断言も積極的に使う） |
| 絵文字 | 使わない |
| ハッシュタグ | 使わない |
| 文字数の目安 | スレッド①：60〜100文字／スレッド②③：500〜800文字 |

---

## 声のトーン（ちほさんの分身として）

- **凛としていて、温かい**：押しつけず、ただそこにいる。静かに寄り添う専門家
- **断言するが、裁かない**：「〜です」と言い切りながら、誰も責めない
- **自分ごととして書く**：「〜な人は〜しがち」という外部分析は絶対にしない。読者の体験をそのまま描写する
- **余白を作る**：「〜かもしれません」「〜てきたかもしれません」で読者に受け取る余地を残す

---

## 文体の特徴

- **です・ます調を基本**にしつつ、体言止めや短い断言文を積極的に織り交ぜる
  - 例：「正論では、人を動かすことはできません。」「でも、あなたはゴミ箱じゃない。」
- **逆説・否定から入る**：「〜ではありません」「〜することでもありません」で読者の思い込みを静かに崩す
- **感情はラベルでなく言葉のまま置く**：「不安」ではなく「寂しかった。わかってほしかった。」
- **箇条書きは「・」**：具体例を3つ程度並べる。前後に空行を入れる

---

## 改行・空白のルール

- 基本は**1〜2文で改行**。長い文は途中でも改行する
- 感情や核心的な一文は**1行単独**に置く
  - 例：「ただ、大切にされたい。安心したい。つながりを感じたいだけ。」
- 箇条書きの前後は**空行**を入れる
- 意味のまとまりの区切りに空行を入れ、読む呼吸を作る

---

## 投稿順のルール

- 同じテーマは**3投稿以上空けてから**繰り返す
  - 例：A → B → D → A はOK　／　A → B → A はNG
- Aテーマ（逆説・否定形）はリーチが高いが、連続させると飽きられる。間隔を意識して使う
- **投資・行動教育型（Eテーマ）は週1〜2本必須**で既存投稿の合間に入れる

---

## テーマ別パフォーマンス（優先度の参考）

高リーチ投稿の分析から、テーマ別に以下の傾向がある。

| テーマ | 代表リーチ | 優先度 |
|--------|-----------|--------|
| A. 逆説・否定形（絶対に〜しない） | 61,000 / 42,000 / 39,000 | 最優先 |
| B. 原因・ロジック解説（なぜ上手くいかないか） | 38,000 / 31,000 / 27,000 | 高 |
| D. 手放し・引き算（良い妻を卒業する） | 38,000 / 14,000 / 13,000 | 高 |
| C. 習慣・マインド（内側の在り方） | 19,000 / 17,000 / 14,000 | 通常 |
| E. 投資・行動教育（自己投資・決断・先延ばしコスト） | — | 週1〜2本必須 |

Aテーマが安定して最も高いリーチを出している。新規投稿はAテーマを軸に企画することを推奨する。

## Threads投稿の構成（スレッド分割型）

### スレッド①：冒頭ティーザー（答えを明かさず続きへ）

**型：**
```
〔テーマ〕な妻が〇〇していること。
それは、〜でも
〜でもありません。
〔動詞句〕のは、
```

- スレッド①の締め言葉は `しないのは、` `手放したのは、` `やめたのは、` `一番大きいのは、` など、テーマに合った動詞句で終わる
- `実は、` は使わない
- 答えはスレッド②の冒頭で明かす
- **淡々とした事実の提示**から始める。問いかけ禁止
- 「〜ではありませんか？」は絶対に使わない

**実例：**
```
夫婦仲がいい妻が、絶対にしないこと。
それは、感情的にならないことでも、
不満を言わないことでもありません。
しないのは、
```

---

### スレッド②：本編（心理的な旅）／ スレッド③：CTA専用

**最重要原則：スレッド②とスレッド③は役割が完全に分かれている。**

- **スレッド②** = 読者を「心理的な旅」に連れていく本編。ここで完全に腑に落とす
- **スレッド③** = CTAのみ。`とはいえ、` から始まりガイドブック誘導で終わる

スレッド②で腑に落とし切れていないままスレッド③に渡してはいけない。

---

### スレッド②：感情の動線で展開する

#### ▼ スレッド②冒頭（最重要フック）

スレッド①の動詞句を受けて、カギカッコで「答え」を置く。
**これが最も読まれる一文。**短く、体言止めで、日常の言葉で書く。前後に空行を入れて際立たせる。

**スレッド②冒頭には3つの型がある（CSVデータ分析より）：**

**型①：「大切なものを、意外な方法で扱っている」構造**
読者が「そんなつもりじゃなかった」と気づく。
```
「心の平安を、夫に委ねること」          ← リーチ42,000
「安心を、夫の態度に委ねること」        ← リーチ39,000
「夫の機嫌を、今日の自分の価値に変換すること」  ← リーチ61,000
```

**型②：「〜という前提」の暴露**
読者が意識していない信念を言語化する。LINE登録が最も増えた2投稿はどちらもこの型。
```
「私は大切にされていない」前提で、夫と関わること  ← LINE+13・フォロワー+42
「夫が変われば、私は満たされる」という前提      ← LINE+12
```

**型③：「無意識の行動」の名付け**
読者がやっている行動に名前をつける。
```
「助けて」を飲み込んだまま自分でなんとかしてしまうこと  ← リーチ20,000
「自分をすり減らす関わり方」
```

**型④：「実践・問い」型（CテーマまたはDテーマで使う）**
スレッド①が「密かにやっていること」「習慣にしているのは、」など**肯定形**の締め言葉のとき、答えは「読者が向かう先」になる。
```
「自分の機嫌を、自分で整えること」  ← リーチ19,000（実践習慣）
「世界の中心を、自分に戻すこと」    ← リーチ21,000（向かう方向）
「私が今、何を感じているか」        ← リーチ17,000（読者自身への問い）
```
型①②③が「読者がやってしまっていること（ネガティブな行動・前提）」を名付けるのに対し、型④は「読者が向かう先（ポジティブな習慣・問い）」を提示する。Aテーマには使わない。

**共通ルール：**
- 説明しない。断言する
- 「〜すること」「〜という前提」で体言止めにする
- 「こうなっている状態」を説明するのはNG（低リーチの共通パターン）

---

#### ▼ スレッド②③の感情の動線（展開の順番）

**論理で説明するのではなく、感情の流れで展開する。**
読者の内側を順番に動かしていく。

```
① 具体的な場面を短い文で積み上げる（3〜5行）
② 読者の内心の声をカギカッコで置く
③ 「でもそれは〜ではありません」で責任を外す
④ 心の仕組み・前提として名付ける（「〜だけ」で断言）
⑤ 柔らかい本音を掘り出す（「本当は〜だけ」）
⑥ 夫婦仲がいい妻は〜（2〜3行で完結）
⑦ 実践ヒント（たとえば〜）
⑧ とはいえ → CTA
```

**① 具体的な場面を積み上げる**
読者の「昨日・今朝」にあった場面を、感情を煽らずそのまま置く。
```
（例）
夫が黙って帰ってくる。
ため息をつく。
返事が短い。
```

**② 内心の声をカギカッコで置く**
読者が心の中で言っていた言葉をそのまま引用する。
```
（例）
「私、何かした？」
「私のせいかな」
「空気を変えないと」
```

**③ 「でもそれは〜ではありません」で責任を外す**
```
（例）
でもそれは、
あなたの我慢が足りないからでも
優しさが不足しているからでもありません。
```

**④ 心の仕組み・前提として名付ける**
短く、体言止めか「〜だけ。」で締める。
```
（例）
夫の状態で自分の安心を確かめようとしてきただけ。
```

**⑤ 柔らかい本音を掘り出す**
怒りや不満の奥にある本当の気持ちを言葉にする。高リーチ・高LINE登録に共通するパターン。
```
（例）
本当は、
ただ大切にされたかった。
安心したかっただけ。
```

**⑥ 夫婦仲がいい妻は〜**
理想を押しつけず、対比として静かに提示する。2〜3行で完結させる。
その後、**鋭く短い1行でinsightを結晶化する**。これが読者の心に残る一文になる。

```
（例）
夫婦仲がいい妻は、
夫が無口な日も、
それを自分への評価にしません。
夫の沈黙は、夫の沈黙。    ← ここがinsight結晶化（体言止め・対称構造）
自分の状態は、自分で決める。
```

**insight結晶化のルール：**
- 「夫婦仲がいい妻は〜」の後に、1行で概念を言い切る
- 長い説明はしない。短くて、パッと頭に入る言葉にする
- 「AはA」（同語反復）「Bは自分で決める」（主権宣言）の形がよく使われる

---

#### ▼ 接続パターン（感情の緊張→共感の流れを作る）

展開の途中で、以下の接続語を使うと「読者がうなずく瞬間」を作れる。
論理的な接続詞（なぜなら・だから・しかし）は使わない。

| 接続パターン | 効果 | 使い所 |
|------------|------|--------|
| `それでも、なぜか〜` | 「わかってるのにできない」共感 | 場面描写→内心の声の間 |
| `そうやって〜` | 習慣化・蓄積の描写 | 繰り返されてきたことを示す |
| `いつの間にか〜` | 無意識の積み重ね | 「気づかないうちに」と同義 |
| `気づかないうちに〜` | 読者が自分ごとにできる | 前提・癖の名付けの前 |

```
（例）2026-05-06_夫との距離.md より
それでも、なぜか届かない。
関係を壊さないようにしてきたのに、
距離が埋まらない。
```

---

#### ▼ 原因の解放（2段階で行う）

「あなたのせいではない」を1回だけ言うと、読者は「でも自分が悪いのかも」と戻ってしまう。
**外責め→自責め、両方を順番に外す**ことで、完全に解放される。

```
【第1段階：心理的な仕組みのせいにする（外責め・自責め両方をいったん保留）】
これは、
気にしすぎているからでも
心が弱いからでもありません。
「夫の沈黙＝自分が原因」という前提が、
無意識に動いているだけ。

【第2段階：さらに深い前提を外す（完全解放）】
でもそれは、
あなたが悪いからでも
関係がうまくいっていないからでもありません。
ただ、
夫の状態で自分の安心を
確かめようとしてきただけ。
```

**ポイント：**
- 第1段階で「仕組み・前提」に帰属させる
- 第2段階で「それも、あなたが悪いからではない」と完全に解放する
- 「ただ、〜してきただけ。」で着地させると柔らかくなる

## CTAへのブリッジ構造

### パターンA：ガイドブック誘導（基本形）

```
とはいえ、

長年の〜は、
そう簡単には抜けませんよね。

〔なぜそうなるのか・どこから整えるのか〕という問いを入れる。

〔解決策の一言〕を
ガイドブックにまとめました。
〔対象者〕は、プロフィールから受け取ってください。
→
```

**重要：「なぜ〜なのか」の問いをCTA前に入れる**
LINE登録が最も多かった投稿（3/30）のパターン。
読者が「それが知りたい」と感じてプロフィールに進む。
```
（例）
なぜそう見えてしまうのか。
どうすればその前提を外せるのか。
ガイドブックにまとめています。
```

**ルール：**
- `とはいえ、` はひらがな表記で統一（`とは言え、` は使わない）
- `とはいえ、` の後は必ず空行
- 「長年の癖」「長年の前提」への共感を必ず入れる
- 「ガイドブック」という言葉を使う
- 最後は「→」で締める
- 投稿内に「LINE」「ミニセッション」の言葉は出さない

**実例（高LINE登録パターン）：**
```
とはいえ、

長い間「外側」に求めてきたものを、
自分に戻すことは簡単ではありません。

なぜそう見えてしまうのか。
どうすればその前提を外せるのか。
ガイドブックにまとめています。

プロフィールから受け取れます。
→
```

## E型：投資・行動教育型の投稿（週1〜2本必須）

既存の夫婦仲テンプレの合間に週1〜2本入れる。リーチよりもLINE登録・フォロワー獲得への効果が高い。

**目的：** 「自分の人生を変えるためにお金と時間を使う価値観」を持った読者を惹きつける。

**3つのパターン（2026/5/8ミーティング資料より）：**

| パターン | 内容 | 使用するPDFポスト案 |
|--------|------|----------------|
| 自己選択・投資誘導型 | 「いつか」では変わらない。先に決める人が変わる | ①③ |
| 先延ばしコスト警告型 | 待ち続けることで何を失っているか | ④⑨ |
| 自己投資の正当化型 | 「自分に使うのは贅沢」という前提を崩す | ⑦⑧ |

**E型の構造（夫婦仲テンプレと同じ型を使う）：**
```
スレッド①：既存テンプレ型で入る
  ↓
スレッド②冒頭：「手放したのは、〇〇という過信/前提/思い込み」
  ↓
展開：後回し→我慢→消耗の感情の流れで見せる
  ↓
再定義：「自己投資してください」とは言わず、
        「自分を後回しにしない感覚を取り戻すこと」として着地させる
  ↓
CTA：ガイドブック誘導またはフォロー誘導
```

**週間企画での表記ルール：**
E型投稿を提案する際は、使用したPDFポスト案の番号と型名を必ず明記する。
例：`[E・④決断の回数型]`

## NG表現チェックリスト

投稿前に確認する。

- [ ] 冒頭が「〜ではありませんか？」の問いかけになっていない
- [ ] 「〇〇な人は〜しがち」という外部分析になっていない
- [ ] 「夫が悪い」「相手が変われば」という構図になっていない
- [ ] 評価語（ダメ・良くない・すべき）を使っていない
- [ ] 専門用語（投影、メタ認知、外在化）をそのまま使っていない
- [ ] 絵文字を使っていない
- [ ] ハッシュタグをつけていない
- [ ] 「本気の方だけ」「〇〇できない人は」という選別をしていない
- [ ] CTAで「LINE」「ミニセッション」という言葉を使っていない
- [ ] 「〜してください」という指示でCTAを終えていない
- [ ] `とはいえ、` はひらがな表記になっている
- [ ] スレッド②冒頭が「状態の説明」になっていない（断言・名付けになっているか）
- [ ] 感情の動線（場面→内心の声→名付け→本音→対比）の順番になっているか
- [ ] 「なぜなら」「つまり」など説明語で展開していない
- [ ] CTA前に「なぜ〜なのか」の問いが入っているか

## 低リーチになるパターン（NG構造）

CSVデータ分析から導出。以下の構造は低リーチと相関する。

- スレッド②冒頭が「こうなっている状態を説明」している
  - NG例：「これは、相手の感情まで自分が背負ってしまう心の癖です。」（説明から入っている）
  - OK例：「夫の機嫌を、今日の自分の価値に変換すること」（行動を名付けている）
- 「なぜなら〜」「だから〜」という論理展開が前に出ている
- 読者の体験描写をせずに結論から入る
- 比喩（「器が空になる」など）で抽象化する
