# Threads 違和感パターン集（学習ログ）

> **目的**: Claudeの下書きにちほさんが感じた「違和感」を1行ずつ蓄積し、同じ違和感を二度と出さないための学習ファイル。振り返りシートの「修正パターン集」と同じ仕組み。
> **運用**: ちほさんは文章を直さなくてよい。「②の3行目、説明っぽい」「この言葉は使わない」のような**短い指摘だけ**もらい、書き直しはClaudeがやる。指摘のたびに、Claudeがここへ1行追記する。
> **Claudeへ**: 新しい投稿を書く前に、必ずこのファイルを全部読むこと。回を重ねるごとに指摘が減ることが、この仕組みの成否の指標。

## 違和感の記録

| 日付 | 直された箇所（要約） | 違和感の種類 | 今後のルール |
|------|--------------------|-------------|-------------|
| （例）| スレッド②の締めが「〜してみませんか？」だった | 提案の押しつけ感 | 提案形で締めない。事実か本音の言葉で終える |
| | | | |

## 違和感の種類（分類の目安）

1. **説明っぽい**（講義になっている。読者の体験描写になっていない）
2. **きれいすぎる**（整いすぎて、ためらい・生っぽさがない）
3. **言葉が違う**（ちほさんが使わない言い回し・ラベル語）
4. **深読みしすぎ**（読者の心理を決めつけている）
5. **構成・長さ**（回りくどい／薄い／リズムが悪い）

## 立ち上げ期の運用（最初の2〜3本）

最初の数本は「調整期間」と割り切る。1本目から完璧は狙わず、**ちほさんの違和感を集めることを目的**にする。
下書き提示の際、Claudeは末尾に「違和感があった行の番号だけ教えてください」と添える。
