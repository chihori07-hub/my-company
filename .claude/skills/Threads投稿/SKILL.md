# Threads投稿作成

毎日のThreads投稿を作成する。スタイルガイドは必ず読み込んでから作業する。

**スタイルガイド：** `/home/user/my-company/.company/marketing/threads-style-guide.md`
**過去の反応が良かった投稿：** `/home/user/my-company/.company/marketing/campaigns/反応が良かったThreads投稿とテーマ.pdf`

---

## モードの判定

`$ARGUMENTS` を確認する。

- **引数なし** または **「週間テーマ」** → 週間テーマ企画モード
- **テーマが渡された** → 投稿下書きモード

---

## モード①：週間テーマ企画

1週間分（7日）のThreadsテーマを提案する。

### 手順

1. スタイルガイドのテーマ別パフォーマンス表を参照する
2. 以下のルールでテーマを組む
   - Aテーマ（逆説・否定形）を週2〜3回を軸に使う
   - 同じテーマは3投稿以上空けてから繰り返す
   - **Eテーマ（教育・投資型）を週1〜2本、A〜Dの合間に挟む**（5〜6月の重点運用期間）
   - 例：A→B→E→A→D→C→E のような配置
3. 7日分を以下の形式で提案する。スレッド①冒頭はほぼ固定テンプレートのため、各日のスレッド①冒頭行（数語の変更のみ）を直接記載する

```
月：[テーマ種別] スレッド①冒頭行
火：[テーマ種別] スレッド①冒頭行
水：[テーマ種別] スレッド①冒頭行
木：[テーマ種別] スレッド①冒頭行
金：[テーマ種別] スレッド①冒頭行
土：[テーマ種別] スレッド①冒頭行
日：[テーマ種別] スレッド①冒頭行
```

4. 確認・修正をもらい、確定したら `marketing/content-plan/` に保存する

---

## モード②：投稿下書き

渡されたテーマで スレッド①②③ の下書きを作成する。

### 手順

1. スタイルガイドを読み込み、テーマ種別（A/B/C/D/E）を確認する
2. スレッド①②③ を生成する

#### スレッド①（60〜100文字）
- 冒頭ティーザー。答えを明かさず続きへ誘う
- 淡々とした事実の提示から始める。問いかけ禁止
- 「〜ではありませんか？」は絶対に使わない
- 「実は、」は使わない
- 締め言葉は `しないのは、` `手放したのは、` `やめたのは、` `一番大きいのは、` などテーマに合った動詞句で終わる

#### スレッド②③（合計500〜800文字）

**A〜Dテーマの場合：**

スレッド②は必ず「答え」から始める（カギカッコで1〜2行、前後に空行）。
その後、以下の5ステップで展開する：

- **Step1** 行動を淡々と描写（「〜かもしれません」で余地を残す）
- **Step2** 心の癖・無意識の前提として名付ける
- **Step3** 「でもそれは〜ではありません」で責任を外す
- **Step4** 本当の原因を1行で断言（体言止めか「〜だけ。」）
- **Step5** 「夫婦仲がいい妻は〜」で対比を静かに提示
- **CTA** パターンA（ガイドブック誘導）またはパターンB（フォロー誘導）で締める

**Eテーマ（教育・投資型）の場合：**

スタイルガイドの「E. 教育・投資型の投稿構成」セクションを必ず参照する。
ステップ構成ではなく「感情の因果関係」の流れ（場面→止まる理由→結果→着地）。
着地は「気づき」で終わらせず「覚悟・選択・行動」まで持っていく。
CTAはパターンA（ガイドブック誘導）またはパターンB（フォロー誘導）どちらも可。
Eテーマ専用NGルール（投資額に触れない、責める表現を使わない等）を遵守する。

### 生成後にNGチェックをしてから提示する

- [ ] 冒頭が問いかけになっていない
- [ ] 「〇〇な人は〜しがち」という外部分析になっていない
- [ ] 「夫が悪い」という構図になっていない
- [ ] 評価語（ダメ・良くない・すべき）を使っていない
- [ ] 専門用語をそのまま使っていない
- [ ] 絵文字・ハッシュタグを使っていない
- [ ] 「本気の方だけ」という選別をしていない
- [ ] CTAで「LINE」「ミニセッション」という言葉を使っていない
- [ ] 「〜してください」という指示でCTAを終えていない
- [ ] `とはいえ、` はひらがな表記になっている
- [ ] 各パートが感情の連鎖として一本の糸になっている（論理だけでなく感情で流れているか）
- [ ] （Eテーマのみ）着地が「気づき」で終わっていない（覚悟・選択・行動まで見せているか）

### 提示・修正・保存

1. スレッド①②③ を提示する
2. 修正をもらいながら磨く
3. 確定したら以下に保存する

**保存先：**

社内フォルダに `.md` ファイルとして保存する。

```
/home/user/my-company/.company/marketing/campaigns/threads/YYYY-MM-DD_テーマ一言.md
```
