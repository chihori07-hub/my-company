# Threads投稿作成

毎日のThreads投稿を作成する。スタイルガイドは必ず読み込んでから作業する。

**スタイルガイド：** `.company/marketing/threads-style-guide.md`
**過去の反応が良かった投稿：** `.company/marketing/campaigns/反応が良かったThreads投稿とテーマ.pdf`

---

## モードの判定

`$ARGUMENTS` を確認する。

- **引数なし** または **「週間テーマ」** → 週間テーマ企画モード
- **テーマが渡された** → 投稿下書きモード

---

## モード①：週間テーマ企画

1週間分（7日）のThreadsテーマを提案する。

### 手順

1. スタイルガイドのテーマ別パフォーマンス表を参照する
2. 以下のルールでテーマを組む
   - Aテーマ（逆説・否定形）を週2〜3回を軸に使う
   - **Eテーマ（投資・行動教育型）を週1〜2本必須で入れる**
   - 同じテーマは3投稿以上空けてから繰り返す
   - A→B→E→D→A のようにバランスよく配置する
3. 7日分を以下の形式で提案する。スレッド①冒頭はほぼ固定テンプレートのため、各日のスレッド①冒頭行（数語の変更のみ）を直接記載する

```
月：[テーマ種別] スレッド①冒頭行
火：[テーマ種別] スレッド①冒頭行
水：[テーマ種別] スレッド①冒頭行
木：[テーマ種別] スレッド①冒頭行
金：[テーマ種別] スレッド①冒頭行
土：[テーマ種別] スレッド①冒頭行
日：[テーマ種別] スレッド①冒頭行
```

4. 確認・修正をもらい、確定したら `marketing/content-plan/` に保存する

---

## モード②：投稿下書き

渡されたテーマで スレッド①②③ の下書きを作成する。

### 手順

1. スタイルガイドを読み込み、テーマ種別（A/B/C/D/E）を確認する
2. スレッド①②③ を生成する

#### スレッド①（60〜100文字）
- 冒頭ティーザー。答えを明かさず続きへ誘う
- 淡々とした事実の提示から始める。問いかけ禁止
- 「〜ではありませんか？」は絶対に使わない
- 「実は、」は使わない
- 締め言葉は `しないのは、` `手放したのは、` `やめたのは、` `一番大きいのは、` などテーマに合った動詞句で終わる

#### スレッド②③（合計500〜800文字）

**スレッド②とスレッド③は役割が完全に分かれている。**
- スレッド② = 読者を「心理的な旅」に連れていく本編。ここで完全に腑に落とす
- スレッド③ = CTAのみ。`とはいえ、` から始まりガイドブック誘導で終わる

スレッド②は必ず「答え」から始める（カギカッコで1〜2行、前後に空行）。
その後、スタイルガイドの「感情の動線8ステップ」に従って展開する：

1. 具体的な場面を短い文で積み上げる（3〜5行）
2. 読者の内心の声をカギカッコで置く
3. 「でもそれは〜ではありません」で責任を外す（**2段階で行う**）
4. 心の仕組み・前提として名付ける（「〜だけ。」で断言）
5. 柔らかい本音を掘り出す（「本当は〜だけ」）
6. 「夫婦仲がいい妻は〜」で対比を提示 → **1行でinsightを結晶化する**
7. 実践ヒント（たとえば〜）
8. （スレッド③）とはいえ → CTA

**E型の場合：** スタイルガイドの「E型：投資・行動教育型」セクションと「模範例③」を参照する。使用したPDFポスト案の番号と型名を提案に明記する。

- **CTA** パターンA（ガイドブック誘導）またはパターンB（フォロー誘導）で締める

### 生成後にNGチェックをしてから提示する

- [ ] 冒頭が問いかけになっていない
- [ ] 「〇〇な人は〜しがち」という外部分析になっていない
- [ ] 「夫が悪い」という構図になっていない
- [ ] 評価語（ダメ・良くない・すべき）を使っていない
- [ ] 専門用語をそのまま使っていない
- [ ] 絵文字・ハッシュタグを使っていない
- [ ] 「本気の方だけ」という選別をしていない
- [ ] CTAで「LINE」「ミニセッション」という言葉を使っていない
- [ ] 「〜してください」という指示でCTAを終えていない
- [ ] `とはいえ、` はひらがな表記になっている

### 提示・修正・保存

1. スレッド①②③ を提示する
2. 修正をもらいながら磨く
3. 確定したら以下に保存する

**保存先：**

```
.company/marketing/campaigns/threads/YYYY-MM-DD_テーマ一言.md
```
