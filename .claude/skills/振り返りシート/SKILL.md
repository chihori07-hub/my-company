# 振り返りシート作成

セッションの文字起こしをもとに、クライアントへの振り返りシートを作成する。

## 手順

**ステップ1：文字起こしを読み込む**
`$ARGUMENTS` にファイルパスが指定されている場合はそのファイルを読み込む。指定がない場合はユーザーに文字起こしの貼り付けまたはファイルパスを求める。

**ステップ2：チャット上にテキストで書き出す**
以下のセクション構成に沿って内容を整理し、チャット上に提示する（この時点ではdocxを作らない）。

1. セッション冒頭でお話ししたこと
2. 本日のワーク内容と気づき
3. セッションで整理されたこと・新しい捉え方
4. 日常での継続事項と次回予定（本編のみ）
5. 私からのメッセージ

### 各セクションの書き方ガイド

- **セクション1（冒頭）**：クライアントが話した事実を整理。コーチの解釈は控えめに。感情の状態にも触れる。2〜4段落。
- **セクション2（ワーク内容と気づき）**：ワークの目的・取り組んだ内容・気づきを整理。クライアント自身が気づいたことを中心に。気づきが複数ある場合は小見出しで分ける。
- **セクション3（新しい捉え方）**：コーチからの視点や問いかけで広がった理解を整理。押しつけにならないトーン（「〜という視点をお伝えしました」）。3〜5ポイント、各1〜2段落。
- **セクション4（継続事項・次回予定、本編のみ）**：「次回までのひとつの実験」を明確に示す。箇条書き中心で簡潔に。取り組んだ感想やワークのスクリーンショットを送ってもらう旨を添える。次回日時を明記する。
- **セクション5（メッセージ）**：クライアントの勇気や変化を具体的に肯定。印象に残った言葉やシーンに触れる。次のステップへの視点のヒントを1〜2つ。4〜8段落、長くなりすぎないよう注意。

### トーン・スタイル
- 「です・ます」調で統一
- クライアントの名前は「〇〇さん」
- 説教・指導のトーンにしない。「気づかれました」「見えてきました」等、クライアントが主語の表現を使う
- いい・悪いのジャッジをしない
- 事実と気づきを丁寧に、温かいトーンで書く
- 同じ表現や言い回しが短い間に繰り返されていないか確認する
- **らしい表現**：一緒に、丁寧に、寄り添う、わたし基準、選び直す、本質、本音、どんな自分がいてもいい
- **らしくない表現（使わない）**：効率化、最速、コスパ、救う、ふわふわ、キラキラ、魔法のような、依存

**ステップ3：確認をもらう**
内容・トーン・追加削除についてユーザーに確認を求める。修正がある場合は修正版を再提示し、再度確認を求める。

**ステップ4：OKが出たらdocxを作成する**
Pythonスクリプトを生成してBashで実行し、docxファイルを作成する。python-docxが未インストールの場合は先に `pip3 install python-docx` を実行する。

クライアントのフォルダが存在しない場合は先に作成する。

### 保存先
```
/Users/higashiyamachihori/my-company/.company/coaching/clients/〇〇さん/〇〇さん_第〇回セッション振り返りシート.docx
```

### docxフォーマット仕様
- フォント：游ゴシック、本文11pt
- 行間：1.15
- 見出し1（セクション番号）：16pt太字、カラー #2E75B6
- 見出し2（小見出し）：13pt太字、カラー #2E75B6
- A4サイズ、上下左右1インチマージン
- セクション間に #2E75B6 の区切り線を入れる
