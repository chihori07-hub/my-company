# 振り返りシート作成・セッション伴走

クライアントセッションの前後を一気通貫でサポートする。**モードA（セッション前の準備メモ）**と**モードB（セッション後の振り返りシート作成）**の2つがある。
※2026-07改訂: 受講生5名×42回分のシート分析（`.company/coaching/program-review/統合分析_プログラムブラッシュアップ提案.md`）を反映。

## モードA：セッション前の準備メモ

「明日〇〇さんのセッション」等と言われたら発動。以下を読み込んでから、チャット上に準備メモを出す。

**読み込むもの**
1. `.company/coaching/program-review/〇〇さん_変化の記録.md`（なければ過去シートから作成する）
2. 直近2回分の振り返りシート（ローカル保管分。なければDriveの一覧から取得）
3. ちほさんから共有されたLINEのやりとり・近況（あれば）

**準備メモの構成**
- **前回までの要約**（3行以内）
- **前回の「実験」と確認ポイント**（何を出したか、冒頭で何を聞くか）
- **変化の地図上の現在地の見立て**（特に5〜8回目は揺り戻し警戒。`.company/coaching/programs/変化の地図_クライアント配布用.md` 参照）
- **今の心理状態の推察**（根拠となる本人の言葉を添える。推測は推測と明記）
- **進行案**: 候補ワーク2〜3個（それぞれの狙いと、扱う時の注意点）。ただし外部で何か起きていたらそちら優先が原則（危機対応はこのプログラムの強み）
- **全体進捗**: 12回中の現在地、残り回数で扱いたいステップ
- **ビフォー指標**（`.company/coaching/programs/ビフォー指標3問_初回運用シート.md` 参照）: 初回の準備メモには「3問を自然に聞く」リマインドを、6回目・12回目の準備メモには「再測定」のリマインドを含める。記録先は各クライアントの変化の記録の冒頭

## モードB：振り返りシート作成

> **⚠️ 品質の前提**: これまでのClaude初稿には、ちほさんが毎回1時間近い修正を加えていた（事実の間違い／解釈のズレ・深読み／トーン／長さ・構成の全種類）。この時間を最小化することがこのスキルの最重要目標。以下の「4つの対策」を必ず守る。

### 初稿品質の4つの対策（必須）

1. **事実**: 文字起こしにある発言だけを書く。文字起こしが不明瞭・自動書き起こしの誤変換で確信が持てない箇所は、**推測で埋めずに下書きの末尾に「🔍確認したい箇所リスト」として質問を付ける**（ちほさんが誤りを探すのではなく、質問に短く答えるだけで済む形にする）。
2. **解釈**: 「誰が言ったか」を厳密に（クライアントの気づき／コーチが伝えた視点を混同しない）。文字起こしに根拠のない心理描写・深読みは書かない。どうしても意味づけが必要なら「〜のように見えました」に留めるか、確認リストに回す。
3. **トーン**: 下書きの前に、**そのクライアントの直近の完成版シート1〜2枚をDriveから読み込み、文体・言葉選びを合わせる**（完成版＝ちほさんの修正済み＝正解のコーパス）。
4. **網羅性・構成**（※これまでのClaude初稿の最大の問題は**内容の不足**だった）: **「漏れなく拾う」が最優先。削るのは一瞬でできるが、足りない内容を探すのは時間がかかる。**
   - セッション内で扱った**すべての話題・エピソード・気づき**を漏らさず載せる（完成版シートの密度が基準。要約で薄めない）
   - ワークの書き出し項目は**全件そのまま**載せる（12個の感謝なら12個全部、メリット/デメリットも全項目。「〜など」で省略しない）
   - 下書きの末尾に**「📋 話題カバー表」**を付ける: 文字起こしに登場した話題を一覧にし、それぞれシートのどのセクションに入れたか／意図的に載せなかったか（雑談等）を明記する。**ちほさんが文字起こしを読み返さなくても、抜けの有無を数十秒で確認できるようにするため**
   - 例外はセクション5（メッセージ）のみ4〜8段落に収める。網羅ではなく厳選の場所

### 修正の学習ループ（必須）

ちほさんから修正が入ったら、確認OK後に `.company/coaching/program-review/シート/修正パターン集.md` に「直された箇所→理由の推定→今後のルール」を1行追記する。**次のシートを書く前に必ずこのファイルを読む。** 回を重ねるごとに修正時間が減っていくことがこの仕組みの成否の指標。

**ちほさんへの依頼の仕方**: 修正はちほさんが文章を書き直すのではなく、「2の後半は言ってない」「ここもっと柔らかく」のような**短い指摘だけ**もらい、書き直しは私がやる。

**ステップ1：文字起こしを読み込む**
`$ARGUMENTS` にファイルパスやDrive URLが指定されていればそれを読む。なければ貼り付けかパスを求める。

**ステップ2：チャット上にテキストで書き出す**（この時点ではドキュメント化しない）。末尾に「🔍確認したい箇所リスト」を付ける（なければ「なし」と明記）。

1. セッション冒頭でお話ししたこと
2. 本日のワーク内容と気づき
3. セッションで整理されたこと・新しい捉え方
4. 日常での継続事項と次回予定
5. 私からのメッセージ

### 各セクションの書き方ガイド

- **セクション1（冒頭）**：クライアントが話した事実を整理。コーチの解釈は控えめに。感情の状態にも触れる。2〜4段落。**前回の「実験」の結果を必ず含める**（できなかった場合も「正直に話してくださったこと自体が一歩」と肯定的に記録する）。
- **セクション2（ワーク内容と気づき）**：ワークの目的・取り組んだ内容・気づきを整理。クライアント自身が気づいたことを中心に。気づきが複数ある場合は小見出しで分ける。**構造のあるワーク（出来事/感情/思考/ルール、欠点⇄プラス面など）は表形式を使う**（見返しやすさに直結する）。
- **セクション3（新しい捉え方）**：コーチからの視点や問いかけで広がった理解を整理。押しつけにならないトーン（「〜という視点をお伝えしました」）。3〜5ポイント、各1〜2段落。
- **セクション4（継続事項・次回予定）**：「次回までのひとつの実験」を明確に。**「宿題」ではなく「実験」と呼び、「できなくても全く問題ありません」を必ず添える**。価値観・欠落感など重いワークを案内する時は**「進まなければ当日一緒にやるので安心してください」を必ず添える**（5名全員が1人では完了できなかった実証がある）。取り組んだ感想やワークを送ってもらう旨と、次回日時を明記。
- **セクション5（メッセージ）**：クライアントの勇気や変化を具体的に肯定。印象に残った言葉やシーンに触れる。次のステップへの視点のヒントを1〜2つ。4〜8段落、長くなりすぎない。

### 42回分の分析から加わったルール

1. **本人の言葉を「 」でそのまま引用する**（各シート最低1箇所）。シートの力の源泉は本人の言葉。
2. **測定可能な変化は数字のまま記録する**（例:「立ち直りが半日→約5分」）。本人が変化を認められない時期の証拠になり、将来のビフォーアフターにもなる。
3. **5〜8回目あたりで停滞・自責（「進んでいなくて申し訳ない」等）が出たら**、メッセージに「浮上してくる感情は変わろうとしている証」「止まっているのではなく準備している時間」の意味づけを必ず入れる。全員に来る山であることが分かっている。
4. **抽象的な指示にしない**。「意識してみてください」で終わらせず、いつ・何を・どうするの行動レベルまで下げる（初期シートへの「何をすればいいか戸惑った」というフィードバックあり）。
5. クライアントの語彙に合わせて言葉の階調を選ぶ（例: 天に返す⇄手放しの宣言、御霊⇄本来の自分、意識したものが拡大⇄脳のRAS）。

### トーン・スタイル
- 「です・ます」調で統一
- クライアントの名前は「〇〇さん」
- 説教・指導のトーンにしない。「気づかれました」「見えてきました」等、クライアントが主語の表現
- いい・悪いのジャッジをしない
- 事実と気づきを丁寧に、温かいトーンで
- 同じ表現や言い回しが短い間に繰り返されていないか確認する
- ダッシュ（——）は1シートに数回まで（2026-07-10 ちほさん確認済み）
- **らしい表現**：一緒に、丁寧に、寄り添う、わたし基準、選び直す、本質、本音、どんな自分がいてもいい
- **らしくない表現（使わない）**：効率化、最速、コスパ、救う、ふわふわ、キラキラ、魔法のような、依存

**ステップ3：確認をもらう**
内容・トーン・追加削除についてちほさんに確認を求める。修正があれば修正版を再提示。**確認なしにドキュメント化しない。**

**ステップ4：OKが出たらドキュメント化して保管する**

1. **Googleドキュメントとして作成**（Drive MCPの `create_file` を使用）。ファイル名: `〇〇さん 第〇回振り返りシート`。可能ならクライアントの既存シートと同じフォルダに配置し、URLをちほさんに渡す（クライアントへの共有はちほさんが行う）
2. **ローカル保管**: `/Users/higashiyamachihori/my-company/.company/coaching/program-review/シート/〇〇さん/第〇回_YYYY-MM-DD.md`（git管理外フォルダ。GitHubには上がらない）
3. **変化の記録を更新**: `.company/coaching/program-review/〇〇さん_変化の記録.md` の表に1行追加（回・日付・主テーマ・転換点）
4. **一覧の更新**: 該当クライアントの「振り返りシート一覧」（Drive）に日付とURLを追記できる場合は追記、できなければちほさんに追記を依頼
