# ミニセッション文字起こしの処理・保管

ミニセッション（個別相談）の文字起こしが共有されたら発動する。「ミニセッションの文字起こし」「個別相談の文字起こし」等の言葉＋添付・パス・Drive URLがトリガー。

## 受け取り方（どれでもOK）

1. チャットへの添付ファイル
2. Driveフォルダ「ミニセッション文字起こし」（ID: 1oQpkocbDLEj3jxDtewgEsbj4Pg9Xei8m）に入れて「〇〇さんの分を処理して」
3. ローカルのファイルパス

## 処理ステップ（1件につき）

**ステップ0: 既存資産の確認**
`.company/coaching/mini-sessions/README.md` と `extracts/` の連番を確認し、次の番号を決める（2026-07時点で#27まで）。

**ステップ1: 抽出シートの作成**
`extracts/NN_名前.md` を既存の#01〜27と同じフォーマットで作成する:
- 基本情報（日付・属性・入口の悩み・結果）
- 生の言葉（カテゴリ別・発言のまま）
- 転換点・効いた介入
- 迷いとちほさんの応答
- 結果・フォロー事項

事実のみ。文字起こしにない解釈は書かない（振り返りシートスキルと同じ原則）。

**ステップ2: 1行記録の追記**
`.company/coaching/mini-sessions/ローンチ期・個別相談記録.md` に1行追記:
`日付｜名前｜結果（成約/保留/見送り/案内せず）｜迷いの分類（①〜⑦）｜一言`
迷いの7分類はハンドブック（`個別相談ハンドブック_v1.md`）セクション2の定義に従う。

**ステップ3: 資産への反映（追記して報告）**
- 新しい生の言葉・語彙 → `声の設計図_お客様の生の言葉.md` の該当カテゴリに追記
- 新しい迷いのパターン・効いた応答・引き際の学び → ハンドブックの該当セクションに追記
- 追記した内容はチャットで報告する（何をどこに足したか）

**ステップ4: 結果別のフォロー提案**
- **成約** → ①`.company/coaching/program-review/〇〇さん_変化の記録.md` の雛形を作成（入口の悩み・ミニセッション番号・ビフォー指標欄）②事例ストーリー対応表への登録は不要（プログラム完了後に検討）
- **保留** → フォロー期日と「保留の質」（前向き/シャッター）を1行記録に明記
- **未成約・保留とも** → 希望があれば、その日のセッションの振り返りシート下書きを作成（未成約でも送るのが再訪の種、が実証済みの型）

## プライバシー（厳守）

- すべての成果物は `.company/coaching/mini-sessions/`（git管理外）に保存。**GitHubには絶対に上げない**
- 実名はgit管理外ファイルにのみ書く。git管理下のファイル（スタイルガイド等）へ転記する時は必ず匿名化

## 定期メンテナンス（5件たまるごと）

新規5件ごとに、ハンドブックの集計（成約率・迷いの分布）を更新し、傾向の変化があれば報告する。ローンチ終了後（8月）に全体を振り返り、次のローンチへの改善点をまとめる。
